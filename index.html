Features


Pipeline is a Maya projects manager tool inspeired by openpipeline.
It was crafted with all my experience in animaiton productions as TD for the past few years.
After getting used to Pipeline, you will find it much easier to manage your own projects and animation teams.

  • Manage all your projects and Maya files from within Maya
  • Takes care of versions and naming conventions so nothing gets lost
  • Asset publishing with versions management
  • Thumbnails
  • Animation mode with published assets bank
  • Collaborate on projects on the cloud, shared drives and multiple platforms
  • Designed especially for creating short films and commercials by small teams
  • Projects have users with permissions, so they are protected from technophobe-animators (not included in the beta)

Installation

You need Gulp installed globally:

$ npm i -g gulp
      
$ git clone [git-repo-url] dillinger
      $ cd dillinger
      $ npm i -d
      $ gulp build --prod
      $ NODE_ENV=production node app

Docker

Dillinger is very easy to install and deploy in a Docker container.

By default, the Docker will expose port 80, so change this within the Dockerfile if necessary. When ready, simply use the Dockerfile to build the image.

cd dillinger
      docker build -t <youruser>/dillinger:latest .

This will create the dillinger image and pull in the necessary dependencies. Once done, run the Docker and map the port to whatever you wish on your host. In this example, we simply map port 80 of the host to port 80 of the Docker (or whatever port was exposed in the Dockerfile):

docker run -d -p 80:80 --restart="always" <youruser>/dillinger:latest

Verify the deployment by navigating to your server address in your preferred browser.
